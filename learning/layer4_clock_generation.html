Layer 4: Clock Generation

PLLs, MMCMs, and phase-shifted clocks. The heartbeat of the test system.

Why Clocks Matter

In the burn-in test system, precise timing is critical:

  • vec_clk - When to apply each test vector
  • Phase-shifted clocks - For PULSE pin timing (T/4, 3T/4)
  • Clock enables - Gate clocks on/off for power control
vec_clk
vec_clk_90

Phase Relationship
vec_clk_90 is delayed by 90° (1/4 period) from vec_clk.
vec_clk_180 is delayed by 180° (1/2 period) - it's just inverted.

Clock Sources in Zynq

┌─────────────────────────────────────────┐ │ ZYNQ PS │ PS_CLK ───────────────────►│ │ (33.33 MHz) │ ┌─────────┐ │ │ │ IO PLL │──► FCLK_CLK0 (100 MHz) ───┼──► To PL │ │ │──► FCLK_CLK1 (200 MHz) ───┼──► To PL │ └─────────┘ │ └─────────────────────────────────────────┘ │ ▼ ┌─────────────────────────────────────────┐ │ ZYNQ PL │ │ │ │ FCLK_CLK1 (200 MHz) │ │ │ │ │ ▼ │ │ ┌─────────┐ │ │ │ MMCM │──► vec_clk (variable) │ │ │ │──► vec_clk_90 │ │ │ │──► vec_clk_180 │ │ └─────────┘ │ │ │ └─────────────────────────────────────────┘

Clock Flow

  1. PS_CLK (33.33 MHz) - External crystal
  2. IO PLL - PS generates FCLK_CLK1 (200 MHz)
  3. MMCM in PL - Generates vec_clk and phase-shifted versions

MMCM vs PLL

Xilinx 7-series has two clock management primitives:

FeaturePLLMMCM
Outputs67
Phase shiftFixed onlyFixed + Dynamic
Fractional divideNoYes
JitterLowerSlightly higher
Use caseSimple clock genComplex timing
We Use MMCM
Because we need precise phase shifts (0°, 90°, 180°) for PULSE pin timing.

2016 Implementation (from top.v)

The reference design instantiates clocking in the top module:

// From reference/kzhang_v2_2016/top.v (simplified)

// MMCM for vector clock generation
MMCME2_ADV #(
    .BANDWIDTH("OPTIMIZED"),
    .CLKFBOUT_MULT_F(5.0),      // VCO = 200 * 5 = 1000 MHz
    .CLKOUT0_DIVIDE_F(10.0),    // vec_clk = 1000/10 = 100 MHz
    .CLKOUT1_DIVIDE(10),        // vec_clk_90 = 100 MHz
    .CLKOUT1_PHASE(90.0),       // 90 degree phase shift
    .CLKOUT2_DIVIDE(10),        // vec_clk_180 = 100 MHz
    .CLKOUT2_PHASE(180.0),      // 180 degree phase shift
    .DIVCLK_DIVIDE(1),
    .CLKIN1_PERIOD(5.0)         // 200 MHz input = 5ns period
) mmcm_inst (
    .CLKIN1(FCLK_CLK1),         // 200 MHz from PS
    .CLKFBIN(clk_fb),
    .CLKFBOUT(clk_fb),
    .CLKOUT0(vec_clk),          // 0° phase
    .CLKOUT1(vec_clk_90),       // 90° phase
    .CLKOUT2(vec_clk_180),      // 180° phase
    .LOCKED(mmcm_locked),
    .RST(~FCLK_RESET0_N)
);

How PULSE Pins Use Clocks

Remember PULSE_PIN from Layer 3? It needs phase-shifted clocks:

vec_clk: ____/````````\________/````````\________ │ │ │ │ 0° 180° 0° 180° vec_clk_90: ________/````````\________/````````\____ │ │ │ │ 90° 270° 90° 270° PULSE output: ____/````````````````\________________/` ▲ ▲ │ │ Rise at 0° Fall at 180° (vec_clk edge) (vec_clk_180 edge) For PULSE_PIN: - Rising edge triggered by vec_clk (0°) - Falling edge triggered by vec_clk_180 (180°) - Result: 50% duty cycle pulse, edges at T/4 and 3T/4 
// Simplified PULSE logic (from io_table.v concept)
always @(posedge vec_clk or posedge vec_clk_180) begin
    if (vec_clk)
        pulse_out <= 1'b1;       // Rise at 0°
    else if (vec_clk_180)
        pulse_out <= 1'b0;       // Fall at 180°
end

// Or using vec_clk_90 directly:
assign pulse_out = vec_clk_90;  // 90° shifted = T/4 to 3T/4 high

AI Training Context

LAYER: 4 - Clock Generation
CONCEPTS:
  - MMCM: Mixed-Mode Clock Manager (Xilinx primitive)
  - PLL: Phase-Locked Loop (simpler alternative)
  - Phase shift: Delay clock by degrees (90° = 1/4 period)
  - Clock gating: Enable/disable clock to save power
  - VCO: Voltage-Controlled Oscillator (internal to MMCM)

ZYNQ_CLOCKS:
  PS_CLK: 33.33 MHz (external crystal)
  FCLK_CLK0: 100 MHz (AXI bus clock)
  FCLK_CLK1: 200 MHz (vector timing base)

MMCM_PARAMETERS:
  CLKFBOUT_MULT_F: VCO multiplier (VCO = input * mult)
  CLKOUTn_DIVIDE: Output divider (output = VCO / divide)
  CLKOUTn_PHASE: Phase shift in degrees
  VCO_RANGE: 600 MHz - 1200 MHz (7-series)

PHASE_USAGE:
  vec_clk (0°): Main vector clock, data sampling
  vec_clk_90 (90°): PULSE pin timing (T/4 delay)
  vec_clk_180 (180°): Inverted clock, falling edges

2016_IMPLEMENTATION:
  File: reference/kzhang_v2_2016/top.v
  Approach: MMCME2_ADV instantiated directly in top
  Clocks: vec_clk, vec_clk_90, vec_clk_180, pll_clk0-3

2026_IMPLEMENTATION:
  File: rtl/clk_gen.v (DONE)
  Firmware: firmware/src/regs.rs ClkCtrl
  Approach: MUX-based frequency selection (ONETWO pattern)
  Status: COMPLETE - glitch-free switching via BUFGMUX

QUESTIONS_AI_SHOULD_ANSWER:
  Q: Why use MMCM instead of PLL?
  A: MMCM supports dynamic phase shifting and fractional
     dividers. Needed for precise 90° phase shift.

  Q: How do you calculate MMCM parameters?
  A: VCO = input_freq * CLKFBOUT_MULT_F
     output_freq = VCO / CLKOUTn_DIVIDE
     Must keep VCO in 600-1200 MHz range.

  Q: Why gate the vector clock?
  A: Power savings. When not running vectors, disable
     the clock to reduce switching activity.
