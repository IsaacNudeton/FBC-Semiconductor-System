Layer 5: Vector Engine

How test patterns flow from memory to pins. The core execution unit.

Vector Data Flow

┌─────────────────────────────────────────────────────────────────────────────┐ │ COMPLETE DATA PATH │ │ │ │ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────────┐ ┌──────┐ │ │ │ DDR │ │ DMA │ │ FIFO │ │ Vector │ │ I/O │ │ │ │ Memory │───►│Controller│═══►│ Buffer │───►│ Engine │───►│Cells │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ │ vectors │ │ AXI HP0 │ │ 256x288 │ │ decode │ │ 160 │ │ │ │ stored │ │ 64-bit │ │ entries │ │ execute │ │ pins │ │ │ └──────────┘ └──────────┘ └──────────┘ └──────────┘ └──────┘ │ │ │ │ │ │ │ ARM configures │ │ │ └──────────────────────────────────────────────►│ │ │ start address, results │ │ length, go status │ └─────────────────────────────────────────────────────────────────────────────┘

Step by Step

  1. ARM loads vectors into DDR memory
  2. ARM configures DMA: source address, length
  3. ARM writes "GO" to control register
  4. DMA streams vectors to FIFO via AXI HP
  5. Vector Engine pulls from FIFO on each vec_clk
  6. I/O cells drive/compare based on vector data
  7. Errors counted and logged

Vector Format

2016 Format (288 bits per vector)

┌────────────────────────────────────┐
│  REPEAT [287:256] - 32 bits       │
│  How many times to repeat          │
├────────────────────────────────────┤
│  EXPECTED [255:128] - 128 bits    │
│  What we expect to read            │
├────────────────────────────────────┤
│  DATA [127:0] - 128 bits          │
│  What to drive on pins             │
└────────────────────────────────────┘

Total: 288 bits = 36 bytes per vector

2026 FBC Format (compressed)

┌────────────────────────────────────┐
│  OPCODE [7:0] - 8 bits            │
│  What operation to perform         │
├────────────────────────────────────┤
│  OPERAND [variable]               │
│  Data for the operation            │
└────────────────────────────────────┘

Examples:
  PATTERN_REP: 1 + 36 = 37 bytes
    (opcode + full vector)
  SET_PINS: 1 + 16 = 17 bytes
    (opcode + 128-bit data)
  WAIT: 1 + 4 = 5 bytes
    (opcode + cycle count)
  LOOP_N: 1 + 4 = 5 bytes
    (opcode + iteration count)

Compression: 10-100x smaller
Why FBC Compression Matters

A burn-in test might run millions of vectors. At 36 bytes each:

  • 1M vectors × 36 bytes = 36 MB (raw)
  • 1M vectors with FBC = ~1 MB (typical)

Less memory, less DMA bandwidth, faster loading.

2016 Implementation

From reference/kzhang_v2_2016/top.v:

// Vector streaming from DMA (simplified from top.v)
wire [287:0] vec_data;           // Full vector from FIFO
wire         vec_valid;          // FIFO has data
wire         vec_ready;          // Engine ready for next

// Unpack vector fields
wire [127:0] vec_drive    = vec_data[127:0];
wire [127:0] vec_expected = vec_data[255:128];
wire [31:0]  vec_repeat   = vec_data[287:256];

// Repeat counter
reg [31:0] repeat_count;

always @(posedge vec_clk) begin
    if (vec_valid && vec_ready) begin
        if (repeat_count == 0) begin
            // Load new vector
            repeat_count <= vec_repeat;
            current_drive <= vec_drive;
            current_expected <= vec_expected;
        end else begin
            // Repeat current vector
            repeat_count <= repeat_count - 1;
        end
    end
end

// Ready for next vector when repeat exhausted
assign vec_ready = (repeat_count == 0);

2026 FBC Implementation

Our FBC decoder is more sophisticated - it interprets opcodes:

// From rtl/fbc_decoder.v (simplified)

localparam OP_PATTERN_REP = 8'hB5;
localparam OP_LOOP_N      = 8'hB0;
localparam OP_SET_PINS    = 8'hC0;
localparam OP_WAIT        = 8'hD0;
localparam OP_HALT        = 8'hFF;

reg [2:0] state;
localparam FETCH = 0, DECODE = 1, EXECUTE = 2;

always @(posedge clk) begin
    case (state)
        FETCH: begin
            if (fifo_valid) begin
                opcode <= fifo_data[7:0];
                state <= DECODE;
            end
        end

        DECODE: begin
            case (opcode)
                OP_PATTERN_REP: begin
                    // Next 288 bits are vector + repeat count
                    load_vector <= 1;
                    state <= EXECUTE;
                end

                OP_SET_PINS: begin
                    // Next 128 bits are pin values
                    pins_out <= fifo_data[127:0];
                    state <= FETCH;
                end

                OP_WAIT: begin
                    // Next 32 bits are cycle count
                    wait_count <= fifo_data[31:0];
                    state <= EXECUTE;
                end

                OP_HALT: begin
                    running <= 0;
                    state <= FETCH;
                end
            endcase
        end

        EXECUTE: begin
            // Handle repeats, waits, loops
            if (repeat_count > 0) begin
                repeat_count <= repeat_count - 1;
                // Apply current vector
            end else begin
                state <= FETCH;
            end
        end
    endcase
end

The FIFO Buffer

Between DMA and engine sits a FIFO to decouple the two clock domains:

AXI Clock Domain Vector Clock Domain (100 MHz) (variable, e.g. 50 MHz) │ │ ▼ ▼ ┌──────────────────────────────────────────────┐ │ ASYNC FIFO │ │ ┌────┬────┬────┬────┬────┬────┬────┬────┐ │ │ │ V0 │ V1 │ V2 │ V3 │ V4 │ │ │ │ │ │ └────┴────┴────┴────┴────┴────┴────┴────┘ │ │ ▲ │ │ │ │ write_ptr │ read_ptr │ │ │ │ Depth: 256 entries │ │ Width: 288 bits (or 64 for FBC stream) │ └──────────────────────────────────────────────┘

Why Async FIFO?

  • Clock domain crossing - AXI at 100MHz, vec_clk at variable rate
  • Burst absorption - DMA bursts faster than execution
  • Flow control - Backpressure when FIFO full

QUESTIONS_AI_SHOULD_ANSWER:
  Q: Why use DMA instead of CPU copying?
  A: Speed and CPU freedom. DMA moves data at bus speed
     while CPU does other work. No per-byte overhead.

  Q: What happens if FIFO underflows?
  A: Vector engine stalls, waiting for data. Test timing
     is disrupted. Must size FIFO to prevent this.

  Q: How does FBC compression work?
  A: Instead of raw vectors, send opcodes. LOOP_N can
     represent 1000 vectors with 5 bytes. WAIT replaces
     repeated identical vectors.
