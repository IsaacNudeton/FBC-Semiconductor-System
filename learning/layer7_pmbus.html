Layer 7: PMBus & Hardware Communication

I2C, PMBus protocol, and power supply control

Overview

The burn-in system communicates with power supplies, thermal controllers, and other peripherals via PMBus - a protocol built on top of I2C (SMBus). Understanding this is critical for the bare-metal firmware.

Hardware Interfaces

Pico (Vicor)

Core Power Supply

High-current core voltage supplies. Controlled via DAC + MIO enable pins.

  • 6 channels available
  • DAC sets voltage reference
  • MIO pin enables output

Lynx / ONETWO Thermal

Thermal Controller

Controls heater and fan for DUT temperature. FBC uses ONETWO crystallization instead of Lynx PID.

  • Temperature setpoint
  • ONETWO crystallization (no tuning)
  • Pattern-aware feedforward

See thermal details →

LC/HC Power Supplies

PMBus

12A (LC) and 40A (HC) programmable power supplies on I2C bus.

  • Address: 0x10-0x7F
  • Voltage/current readback
  • On/off control

XADC

FPGA Internal ADC

Built into Zynq FPGA. Monitors temperature and voltages.

  • 12-bit resolution
  • 16 auxiliary channels
  • On-chip temp sensor

External ADC

SPI

32-channel external ADC for DUT measurements.

  • 0V to 3V range
  • ~20ns per sample
  • SPI interface

External DAC

SPI

10-channel DAC for analog output control.

  • 0V to 4.096V range
  • Core voltage reference
  • SPI interface

PMBus Protocol

PMBus is a variant of SMBus designed for power supply control. It uses command codes to read/write parameters.

Common PMBus Commands

Command Code Type Description
OPERATION 0x01 R/W Byte Turn output on/off
ON_OFF_CONFIG 0x02 R/W Byte Configure on/off behavior
VOUT_MODE 0x20 R Byte Output voltage format
VOUT_COMMAND 0x21 R/W Word Set output voltage
READ_VOUT 0x8B R Word Read output voltage
READ_IOUT 0x8C R Word Read output current
READ_TEMPERATURE 0x8D R Word Read temperature
MFR_ID 0x99 R Block Manufacturer ID
MFR_MODEL 0x9A R Block Model name

Linear Data Formats

PMBus uses two linear formats for voltage/current/power values:

Linear11 (5-bit exponent, 11-bit mantissa)

// Used for: READ_IOUT, READ_POUT, etc.
fn linear11_to_float(raw: u16) -> f32 {
    let exp = ((raw >> 11) & 0x1F) as i8;
    let exp = if exp > 15 { exp - 32 } else { exp };  // Sign extend
    let mantissa = (raw & 0x7FF) as i16;
    let mantissa = if mantissa > 1023 { mantissa - 2048 } else { mantissa };

    mantissa as f32 * 2.0_f32.powi(exp as i32)
}

Linear16 (exponent from VOUT_MODE)

// Used for: READ_VOUT, VOUT_COMMAND
fn linear16_to_volts(raw: u16, vout_mode: u8) -> f32 {
    let exp = (vout_mode & 0x1F) as i8;
    let exp = if exp > 15 { exp - 32 } else { exp };  // Sign extend

    raw as f32 * 2.0_f32.powi(exp as i32)
}

fn volts_to_linear16(volts: f32, vout_mode: u8) -> u16 {
    let exp = (vout_mode & 0x1F) as i8;
    let exp = if exp > 15 { exp - 32 } else { exp };

    (volts / 2.0_f32.powi(exp as i32)) as u16
}

Auto-Discovery Implementation

Instead of hardcoded addresses, scan the I2C bus and identify devices:

/// PMBus device types we support
#[derive(Debug, Clone, Copy)]
enum PmbusDeviceType {
    Unknown,
    MPS,        // MPS power modules
    Infineon,   // Infineon/IR power modules
    Vicor,      // Vicor Pico modules
    TI,         // Texas Instruments
}

/// Discovered PMBus device
struct PmbusDevice {
    addr: u8,
    device_type: PmbusDeviceType,
    vout_mode: u8,  // For voltage calculations
}

/// Scan I2C bus for PMBus devices
fn discover_pmbus_devices(i2c: &mut I2c) -> Vec<PmbusDevice> {
    let mut devices = Vec::new();

    // Standard PMBus address range
    for addr in 0x10..0x7F {
        // Try to read MFR_ID (0x99)
        if let Ok(mfr_id) = i2c.read_block(addr, 0x99) {
            let device_type = identify_manufacturer(&mfr_id);

            // Read VOUT_MODE for voltage calculations
            let vout_mode = i2c.read_byte(addr, 0x20).unwrap_or(0);

            devices.push(PmbusDevice {
                addr,
                device_type,
                vout_mode,
            });

            log::info!("Found {:?} at 0x{:02X}", device_type, addr);
        }
    }

    devices
}

fn identify_manufacturer(mfr_id: &[u8]) -> PmbusDeviceType {
    match mfr_id {
        b"MPS" | [0x4D, 0x50, 0x53, ..] => PmbusDeviceType::MPS,
        b"Infineon" | [0x49, ..] => PmbusDeviceType::Infineon,
        b"Vicor" | [0x56, ..] => PmbusDeviceType::Vicor,
        b"TI" | [0x54, 0x49, ..] => PmbusDeviceType::TI,
        _ => PmbusDeviceType::Unknown,
    }
}

I2C Driver for Zynq

/// Zynq I2C registers
const I2C0_BASE: u32 = 0xE000_4000;

struct I2cRegs {
    cr: Reg<u32>,      // 0x00: Control
    sr: Reg<u32>,      // 0x04: Status
    addr: Reg<u32>,    // 0x08: Address
    data: Reg<u32>,    // 0x0C: Data
    isr: Reg<u32>,     // 0x10: Interrupt Status
    xfer_size: Reg<u32>, // 0x14: Transfer Size
    timeout: Reg<u32>, // 0x1C: Timeout
}

impl I2c {
    /// Read a single byte from a PMBus command
    pub fn read_byte(&mut self, addr: u8, cmd: u8) -> Result<u8, I2cError> {
        // Write command byte
        self.regs.addr.write((addr as u32) << 1);  // Write mode
        self.regs.data.write(cmd as u32);
        self.wait_tx_complete()?;

        // Read data byte
        self.regs.addr.write(((addr as u32) << 1) | 1);  // Read mode
        self.regs.xfer_size.write(1);
        self.regs.cr.write(self.regs.cr.read() | (1 << 0));  // Start
        self.wait_rx_complete()?;

        Ok(self.regs.data.read() as u8)
    }

    /// Read a word (2 bytes) from a PMBus command
    pub fn read_word(&mut self, addr: u8, cmd: u8) -> Result<u16, I2cError> {
        self.regs.addr.write((addr as u32) << 1);
        self.regs.data.write(cmd as u32);
        self.wait_tx_complete()?;

        self.regs.addr.write(((addr as u32) << 1) | 1);
        self.regs.xfer_size.write(2);
        self.regs.cr.write(self.regs.cr.read() | (1 << 0));
        self.wait_rx_complete()?;

        let low = self.regs.data.read() as u8;
        let high = self.regs.data.read() as u8;

        Ok((high as u16) << 8 | low as u16)
    }

    /// Write a word (2 bytes) to a PMBus command
    pub fn write_word(&mut self, addr: u8, cmd: u8, value: u16) -> Result<(), I2cError> {
        self.regs.addr.write((addr as u32) << 1);
        self.regs.data.write(cmd as u32);
        self.regs.data.write((value & 0xFF) as u32);       // Low byte
        self.regs.data.write(((value >> 8) & 0xFF) as u32); // High byte
        self.wait_tx_complete()?;

        Ok(())
    }
}

Power Supply Control Example

/// High-level power supply control
struct PowerSupply {
    device: PmbusDevice,
    i2c: I2c,
}

impl PowerSupply {
    /// Set output voltage
    pub fn set_voltage(&mut self, volts: f32) -> Result<(), Error> {
        let raw = volts_to_linear16(volts, self.device.vout_mode);
        self.i2c.write_word(self.device.addr, 0x21, raw)?;  // VOUT_COMMAND
        Ok(())
    }

    /// Read actual output voltage
    pub fn read_voltage(&mut self) -> Result<f32, Error> {
        let raw = self.i2c.read_word(self.device.addr, 0x8B)?;  // READ_VOUT
        Ok(linear16_to_volts(raw, self.device.vout_mode))
    }

    /// Read output current
    pub fn read_current(&mut self) -> Result<f32, Error> {
        let raw = self.i2c.read_word(self.device.addr, 0x8C)?;  // READ_IOUT
        Ok(linear11_to_float(raw))
    }

    /// Turn output on
    pub fn enable(&mut self) -> Result<(), Error> {
        self.i2c.write_byte(self.device.addr, 0x01, 0x80)?;  // OPERATION = ON
        Ok(())
    }

    /// Turn output off
    pub fn disable(&mut self) -> Result<(), Error> {
        self.i2c.write_byte(self.device.addr, 0x01, 0x00)?;  // OPERATION = OFF
        Ok(())
    }
}

Vicor (Pico) Core Supply Control

The Vicor core supplies use a different approach: DAC sets voltage reference, MIO pin enables output.

/// Core supply channel mapping
const CORE_CHANNELS: [(u8, u8, u8); 6] = [
    // (core_num, dac_channel, mio_pin)
    (1, 9, 0),   // Core 1: DAC 9, MIO 0 (directly powered)
    (2, 3, 39),  // Core 2: DAC 3, MIO 39
    (3, 7, 47),  // Core 3: DAC 7, MIO 47
    (4, 8, 8),   // Core 4: DAC 8, MIO 8
    (5, 4, 38),  // Core 5: DAC 4, MIO 38
    (6, 2, 37),  // Core 6: DAC 2, MIO 37
];

struct CoreSupply {
    dac: ExternalDac,
    gpio: Gpio,
}

impl CoreSupply {
    /// Set core voltage and enable
    pub fn set_voltage(&mut self, core: u8, volts: f32) -> Result<(), Error> {
        let (_, dac_ch, mio_pin) = CORE_CHANNELS
            .iter()
            .find(|(c, _, _)| *c == core)
            .ok_or(Error::InvalidCore)?;

        // Vicor uses ~2V DAC output for voltage reference
        // Actual formula may need calibration
        let dac_volts = volts * 0.5;  // Scale factor TBD

        self.dac.write_channel(*dac_ch, dac_volts)?;
        self.gpio.set_high(*mio_pin)?;  // Enable output

        Ok(())
    }

    /// Disable core output
    pub fn disable(&mut self, core: u8) -> Result<(), Error> {
        let (_, _, mio_pin) = CORE_CHANNELS
            .iter()
            .find(|(c, _, _)| *c == core)
            .ok_or(Error::InvalidCore)?;

        self.gpio.set_low(*mio_pin)?;
        Ok(())
    }
}

Key Takeaways

  • PMBus is I2C with standard command codes for power supplies
  • Linear11 format: 5-bit exponent + 11-bit mantissa (for current/power)
  • Linear16 format: 16-bit mantissa, exponent from VOUT_MODE (for voltage)
  • Auto-discovery via MFR_ID command instead of hardcoded addresses
  • Zynq I2C at 0xE000_4000 (I2C0) and 0xE000_5000 (I2C1)
  • Vicor cores use DAC + GPIO, not PMBus
