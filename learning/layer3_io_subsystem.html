What is an I/O Subsystem?

In a burn-in test system, the I/O subsystem is the bridge between the FPGA and the chip being tested (DUT - Device Under Test).

Each pin can be configured to behave differently:

  • Drive - Push a value onto the pin (0 or 1)
  • Compare - Read the pin and check against expected
  • High-Z - Disconnect (tri-state)
Key Insight
The magic is that EACH of the 160 pins can have a DIFFERENT behavior at the SAME time. One pin drives, another compares, another pulses.

The 9 Pin Types

The 2016 system defines 9 pin types using a 4-bit code. Here's the Verilog from vector.vh:

`define BIDI_PIN        4'b0000  // Bidirectional
`define INPUT_PIN       4'b0001  // Input (compare only)
`define OUTPUT_PIN      4'b0010  // Output (drive only)
`define OPEN_C_PIN      4'b0011  // Open collector
`define PULSE_PIN       4'b0100  // Pulse waveform
`define NPULSE_PIN      4'b0101  // Inverted pulse
`define ERROR_TRIG      4'b0110  // Error trigger
`define VEC_CLK_PIN     4'b0111  // Vector clock out
`define VEC_CLK_EN_PIN  4'b1000  // Clock enable out

Let's understand each one...

Why 4 bits?
4 bits = 16 possible values. Only 9 are used, leaving room for future pin types without changing the bus width.

BIDI_PIN (Bidirectional)

The most common pin type. Can both drive and compare based on the output enable (OEN).

BIDI Pin Timing

pin (actual)

Logic:

// When OEN=1: Drive the pin
pin <= dout;

// When OEN=0: Compare (high-Z output)
pin <= 1'bZ;
error <= (din != expected);
Error Detection
Error occurs when OEN=0 (comparing) AND the pin value doesn't match what we expected. This catches manufacturing defects.

PULSE_PIN (Timed Pulse)

Generates a pulse with precise timing. Rising edge at T/4, falling edge at 3T/4 of the vector clock period.

Pulse Pin Timing (T = one vector clock period)
vec_clk: ____/````\____/````\____/````\____ | | | | | | 0 T/4 T/2 3T/4 T ... pulse_out: ____/`````````\__________/`````````\____ ^ ^ | | Rise at Fall at T/4 3T/4 npulse_out: ````\_________/``````````\_________/```` (inverted)

This requires phase-shifted clocks:

  • vec_clk - 0° phase (base clock)
  • vec_clk_90 - 90° phase (T/4 delayed)
  • vec_clk_180 - 180° phase (T/2 delayed)
Why Pulses?
Some chip pins need clock-like signals or edge triggers. PULSE lets you generate them without wasting vector memory on every edge.

OPEN_C_PIN (Open Collector)

Open collector/drain outputs can only pull LOW or float. They rely on external pull-up resistors to go high.

Open Collector Behavior
dout=0: Pin floats HIGH (external pull-up) FPGA: Z ----+---- External: 1 | === Pull-up resistor | VCC dout=1: Pin driven LOW FPGA: 0 ----+---- External: 0 | === Pull-up (overpowered) | VCC 
// Open collector logic
pin_oen <= dout;        // OEN controlled by data
pin_dout <= 1'b0;       // Always drive low when enabled

// Error check: if we're floating (dout=0)
// and pin reads low, something is pulling it down
error <= (!dout) && (din == 1'b0);
Use Case
I2C buses use open-drain. Multiple devices can share a line - anyone can pull low, and it floats high when everyone releases.

Inside io_table.v

The io_table.v file in the reference design decodes the 4-bit pin type into control signals.

// Simplified from reference/kzhang_v2_2016/io_table.v

module io_cell (
    input  [3:0] pin_type,    // Which type?
    input        dout,        // Data to drive
    input        din,         // Data read from pin
    input        expected,    // Expected value
    input        vec_clk,     // Vector clock
    input        vec_clk_90,  // 90° phase

    output reg   pin_oen,     // Output enable
    output reg   pin_dout,    // Actual output
    output reg   error        // Mismatch detected
);

always @(*) begin
    case (pin_type)
        4'b0000: begin  // BIDI
            pin_oen  = oen_from_vector;
            pin_dout = dout;
            error    = !pin_oen && (din != expected);
        end

        4'b0010: begin  // OUTPUT
            pin_oen  = 1'b1;  // Always driving
            pin_dout = dout;
            error    = 1'b0;  // Never compare
        end

        4'b0100: begin  // PULSE
            pin_oen  = 1'b1;
            pin_dout = vec_clk_90;  // 90° phase = pulse
            error    = 1'b0;
        end

        // ... other types
    endcase
end

endmodule
