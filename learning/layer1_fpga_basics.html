Layer 1: FPGA Fundamentals

How programmable silicon actually works. The foundation for everything else.

What is an FPGA?

Field Programmable Gate Array - a chip where YOU define the logic after manufacturing.

Unlike a CPU (fixed instructions) or ASIC (fixed logic), an FPGA contains:

  • Configurable Logic Blocks (CLBs) - can become any logic function
  • Programmable Routing - wires you can connect/disconnect
  • I/O Blocks - interface to the outside world
  • Hard Blocks - BRAMs, DSPs, PLLs (pre-built, efficient)

Look-Up Tables (LUTs)

The core of FPGA logic. A LUT is just a small memory that implements any boolean function.

6-Input LUT (LUT6)

The Zynq 7020 uses 6-input LUTs. It has 64 bits of storage (2^6 = 64).

┌─────────────────────────┐ A ──►│ │ B ──►│ │ C ──►│ 64-bit Memory │──► Output D ──►│ (INIT value) │ E ──►│ │ F ──►│ │ └─────────────────────────┘ Inputs [F:A] select which of the 64 bits to output. The 64 bits (INIT) define the function.

Example: AND Gate

To make a 2-input AND gate, we only care about inputs A and B:

ABOutput (A AND B)
000
010
100
111

INIT value for AND: 64'h8000000000000000 (bit 63 = 1, rest = 0)

Wait, that's for 6 inputs. For 2-input AND with unused inputs tied to 0:

INIT = 64'h8 → binary 1000 → output 1 only when A=1, B=1

Key Insight
A LUT can implement ANY function of its inputs. AND, OR, XOR, MUX, decoder - all just different INIT values. This is why FPGAs are so flexible.

Flip-Flops (FFs)

LUTs are combinational (no memory). FFs add sequential logic - they remember state.

┌─────────────────────────────────────┐ │ SLICE │ │ │ ──────┼──► LUT ──┬──────────────────► (comb out) │ │ │ │ ▼ │ │ ┌─────┐ │ clk ──┼──────►│ FF │──────────────► (reg out) │ └─────┘ │ │ ▲ │ │ │ │ CE ───┼──────────┘ (clock enable) │ RST ──┼──────────┘ (reset) │ │ │ └─────────────────────────────────────┘

Each CLB contains multiple SLICEs. Each SLICE has:

  • 4 LUT6s (can split into 8 LUT5s)
  • 8 Flip-Flops
  • Carry logic (for fast arithmetic)
  • Muxes (for wider functions)
Zynq 7020 Resources
  • 53,200 LUTs
  • 106,400 Flip-Flops
  • 140 Block RAMs (36Kb each)
  • 220 DSP slices

Programmable Routing

LUTs and FFs are useless without connections. The FPGA has a massive switching network.

Why Routing Matters
In our custom toolchain, routing took 9 iterations to resolve conflicts. Real FPGAs have limited wires - you can't connect everything to everything. This is why placement matters.

The Bitstream

All these configurable elements are controlled by SRAM cells. The bitstream programs them.

Bitstream file: ┌─────────────────────────────────────────────────────────┐ │ Header: sync word (AA995566), device ID │ ├─────────────────────────────────────────────────────────┤ │ Frame 0: [101 words] → Config bits for column 0 │ │ - LUT INIT values │ │ - FF init states │ │ - Routing switch settings │ ├─────────────────────────────────────────────────────────┤ │ Frame 1: [101 words] → Config bits for column 1 │ ├─────────────────────────────────────────────────────────┤ │ ... ~9000 frames for Zynq 7020 ... │ ├─────────────────────────────────────────────────────────┤ │ Footer: CRC, startup commands │ └─────────────────────────────────────────────────────────┘ Total: ~4MB for xc7z020

Each bit in the bitstream controls one thing:

  • One LUT INIT bit
  • One routing switch (PIP)
  • One mux select
  • One FF setting

QUESTIONS_AI_SHOULD_ANSWER:
  Q: How many different functions can a 6-input LUT implement?
  A: 2^64 ≈ 18 quintillion (every possible truth table)

  Q: Why do FPGAs use LUTs instead of actual gates?
  A: Flexibility. One LUT can be any gate. Manufacturing one
     universal element is cheaper than many specific ones.

  Q: What limits FPGA clock speed?
  A: Routing delay. Signals must traverse switches. More hops = slower.

COMMON_MISCONCEPTIONS:
  - "FPGAs are slow" → False. Parallelism beats CPUs for many tasks.
  - "FPGAs are just for prototyping" → False. Used in production (network, HFT, ML).
  - "Verilog runs on the FPGA" → False. Verilog is compiled to bitstream. No CPU.
