Layer 0: The Foundations

Everything complex is built from simple pieces. Let's start from the beginning.

1 It All Starts With a Switch

Think about a light switch in your house. Flip it up = light ON. Flip it down = light OFF. That's it. ON or OFF. 1 or 0. Yes or No.

A computer is just billions of tiny switches. Instead of your finger flipping them, electricity flips them. And they flip really, really fast - billions of times per second.

Real World Connection

The chip we're working with (Zynq 7020) has about 1 billion of these tiny switches. If you flipped a light switch once per second, it would take you 31 YEARS to flip that many times. The chip does it every second.
The technical name: Transistor - a switch with no moving parts. Electricity controls electricity.
↓ Switches can make decisions ↓

2 Switches That Make Decisions

What if you connected two switches together and said: "The light only turns ON if BOTH switches are ON"?

Now you have a decision-maker. It's looking at two inputs and making a choice.
Two switches controlling one light: Switch A: OFF ○───┐ ├───► Light: OFF Switch B: OFF ○───┘ Switch A: ON ●───┐ ├───► Light: OFF (still off, need BOTH) Switch B: OFF ○───┘ Switch A: ON ●───┐ ├───► Light: ON ✓ (both on!) Switch B: ON ●───┘

Real World Connection

This is how your phone's lock screen works: Need BOTH the right face AND the right PIN? That's an "AND" decision. Need EITHER fingerprint OR passcode? That's an "OR" decision.
The technical name: Logic Gate - a few switches connected to make a decision. Common ones: AND (need all), OR (need any), NOT (flip it).
↓ Decisions create math ↓

3 ON and OFF = Numbers

We decided to call ON = "1" and OFF = "0".

Now we can do math with switches! How? Count in a different way:

Instead of 0, 1, 2, 3, 4, 5... (which needs 10 symbols)
We use 0, 1, 10, 11, 100, 101... (only needs 2 symbols)
How we normally count: How computers count: 0 (zero) 0 1 (one) 1 2 (two) 10 ← "one 2, zero 1s" 3 (three) 11 ← "one 2, one 1" 4 (four) 100 ← "one 4, zero 2s, zero 1s" 5 (five) 101 ... 255 11111111 (8 switches) The letter "A" = 65 = 01000001 (8 switches in a pattern) A photo = millions of these patterns for each pixel color

Real World Connection

When you send a text message, your phone converts each letter to a pattern of 1s and 0s, sends those electrical ON/OFF signals through the air, and the other phone converts them back to letters.
The technical name: Binary - counting with only 0 and 1. Each 0/1 is called a bit. Eight bits = one byte.
↓ But switches forget instantly... ↓

4 Switches That Remember

Normal switches forget. The moment you stop pushing, they reset.

But what if a switch could HOLD its position? You push it to ON, let go, and it STAYS on until you push it to OFF.

That's memory. A switch that remembers what you told it.
Normal wire: Memory switch: Input ────────► Output Input ──┬──► Output (instant, forgets) │ Clock ─┘ Input changes? Only copies Input when Output changes instantly "Clock" says "NOW!" No memory Then HOLDS that value Like a camera: - Normal wire = mirror (always showing right now) - Memory switch = photograph (captures one moment, keeps it)

Real World Connection

Your computer's RAM is billions of these memory switches. When you're writing a document, each letter is stored in these little "photograph" switches. When power goes off, they forget (that's why you save to disk).
The technical name: Flip-Flop or FF - a switch that remembers 1 bit. Group 32 together = a register that remembers a number.
↓ Now the magic: programmable switches ↓

5 The Cheat Sheet (LUT)

Imagine you have a tiny filing cabinet with 16 drawers.

You give it 4 questions (like 4 yes/no switches), and it looks up the answer in the matching drawer.

The magic: You can PUT any answer in any drawer. So you can make this filing cabinet do ANY logic you want. Want it to be an AND gate? Put the right answers in. Want it to be an OR gate? Change the answers.
A programmable "filing cabinet": 4 Inputs (questions): Which drawer? Answer in drawer: A=0, B=0, C=0, D=0 ───► Drawer 0 ───► [You decide: 0 or 1] A=0, B=0, C=0, D=1 ───► Drawer 1 ───► [You decide: 0 or 1] A=0, B=0, C=1, D=0 ───► Drawer 2 ───► [You decide: 0 or 1] ... A=1, B=1, C=1, D=1 ───► Drawer 15 ───► [You decide: 0 or 1] 16 drawers total, each stores 0 or 1. Change what's IN the drawers = change what the circuit DOES.

Real World Connection

This is like a vending machine. The buttons you press (inputs) select a slot, and whatever is IN that slot (programmed contents) comes out. Different items in the slots = different output.
The technical name: LUT = Look-Up Table. It "looks up" the answer based on inputs. A LUT6 has 6 inputs and 64 drawers.
↓ Thousands of LUTs = an FPGA ↓

6 The FPGA: A Lego Set of Logic

An FPGA is a chip with thousands of these programmable filing cabinets (LUTs) and memory switches (FFs), all connected by wires you can also program.

It's like a giant Lego set. The pieces are already there. You just decide how to connect them to build whatever you want.
FPGA = A grid of programmable blocks + programmable wires ┌─────────────────────────────────────────┐ │ ┌─────┐ ┌─────┐ ┌─────┐ ┌─────┐ │ │ │ LUT │═══│ LUT │═══│ LUT │═══│ LUT │ │ ← Programmable │ │ +FF │ │ +FF │ │ +FF │ │ +FF │ │ wires connect │ └──╬──┘ └──╬──┘ └──╬──┘ └──╬──┘ │ blocks together │ ║ ║ ║ ║ │ │ ═══╬═════════╬═════════╬═════════╬═══ │ │ ║ ║ ║ ║ │ │ ┌──╬──┐ ┌──╬──┐ ┌──╬──┐ ┌──╬──┐ │ │ │ LUT │═══│ LUT │═══│ LUT │═══│ LUT │ │ │ │ +FF │ │ +FF │ │ +FF │ │ +FF │ │ │ └─────┘ └─────┘ └─────┘ └─────┘ │ │ │ │ ↕ ↕ ↕ ↕ │ ← Pins to outside └─────────────────────────────────────────┘ Our chip: 53,200 LUTs + 106,400 FFs (That's a LOT of Lego pieces)
The technical name: FPGA = Field Programmable Gate Array. "Field programmable" = you program it after it's made (not at the factory).

7 The Recipe File (Bitstream)

A bitstream is like a recipe that tells every LUT what to put in its drawers, and every wire whether to connect or not.

It's just a big file of 1s and 0s. Load it into the chip, and the chip BECOMES your circuit.

What We Built

Normally you need Xilinx's software (Vivado, 30GB) to create this recipe file. We built our own toolchain that creates the same recipe, and we learned the format by studying examples (ONETWO method).
The technical name: Bitstream = the configuration file. .bit file extension. Ours is about 3MB for the full design.

Level 1: Switches (transistors) ↓ Level 2: Decisions (logic gates - AND, OR, NOT) ↓ Level 3: Numbers (binary - 1s and 0s) ↓ Level 4: Memory (flip-flops - remembering) ↓ Level 5: Programmable logic (LUTs - the filing cabinet) ↓ Level 6: The chip (FPGA - thousands of LUTs + wires) ↓ Level 7: The recipe (bitstream - tells chip what to become) ↓ Level 8: Your circuit running as real hardware!
