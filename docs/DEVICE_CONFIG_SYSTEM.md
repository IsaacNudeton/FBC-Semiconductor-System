This document describes the device configuration system that converts Sonoma project files to binary format for the FBC controller.

## Tool: fbc-config

**Location:** `tools/fbc-config/`

### Build

```bash
cd tools/fbc-config
cargo build --release
```

### Commands

#### 1. Compile - Convert Sonoma files to binary

```bash
fbc-config compile --bim <file.bim> [--map <file.map>] [--tim <file.tim>] -o <output.fbc>
```

**Inputs:**
- `--bim` (required): XML file defining pins, types, DUT layout
- `--map` (optional): Pin mapping (tester channel → DUT signal)
- `--tim` (optional): Timing configuration (pin types, pulse delays, frequency)

**Output:**
- Binary `.fbc` file ready for controller

**Example:**
```bash
fbc-config compile \
  --bim /path/to/normandy.bim \
  --map /path/to/normandy.map \
  --tim /path/to/debug.tim \
  -o normandy.fbc
```

#### 2. Info - Display binary contents

```bash
fbc-config info <file.fbc>
```

**Output:**
```
FBC Device Configuration: normandy.fbc
  Magic: 0x00434246
  Version: 1
  BIM Type: 31
  Pin Count: 91

Timing:
  freq_sel: 0 (5 MHz)
  delay0: 0
  delay1: 0

Configured Pins:
    0: PWRGD            type=0 (BIDI)
   45: JTAG_TCK         type=4 (PULSE)
  126: JTAG_TDO         type=1 (INPUT)
  ...
```

#### 3. Header - Generate C header for firmware

```bash
fbc-config header <file.fbc> [-o <output.h>]
```

**Output:** C header with register arrays that firmware can directly write to AXI.

```c
// Auto-generated by fbc-config
#define FBC_CONFIG_BIM_TYPE  31
#define FBC_CONFIG_PIN_COUNT 91
#define FBC_CONFIG_FREQ_SEL  0

// Pin type registers (io_config base + 0x000)
static const uint32_t fbc_pin_type_regs[20] = {
    0x00000000,  // pins 0-7
    0x00400000,  // pins 40-47 (pin 45 = PULSE)
    0x11000000,  // pins 120-127 (pins 126,127 = INPUT)
    ...
};

// Pulse control registers (io_config base + 0x200)
static const uint32_t fbc_pulse_ctrl_regs[80] = {
    0x00000000,  // pins 0, 1
    ...
};
```

#### 4. New - Create XML template

```bash
fbc-config new <device_name> [--pins <count>] -o <output.bim>
```

Creates a starter `.bim` file for a new device.

## Binary Format (.fbc)

The binary format maps directly to FPGA AXI registers defined in `rtl/io_config.v`.

### Layout

| Offset | Size (bytes) | Field | Description |
|--------|--------------|-------|-------------|
| 0x0000 | 4 | magic | 0x00434246 ("FBC\0") |
| 0x0004 | 2 | version | Format version (currently 1) |
| 0x0006 | 1 | bim_type | BIM type ID from .bim file |
| 0x0007 | 1 | pin_count | Number of configured pins |
| 0x0008 | 1 | freq_sel | Clock frequency (0-4) |
| 0x0009 | 1 | delay0 | Delay line 0 tap |
| 0x000A | 1 | delay1 | Delay line 1 tap |
| 0x000B | 5 | reserved | Padding |
| 0x0010 | 80 | pin_type_regs[20] | Pin type (4 bits × 160 pins) |
| 0x0060 | 320 | pulse_ctrl_regs[80] | Pulse timing (16 bits × 160 pins) |
| 0x01A0 | 2560 | pin_names[160][16] | Signal names (for GUI) |

**Total size: 2976 bytes**

### Register Mapping

#### Pin Type Registers (20 × 32-bit)

Each register holds 8 pins × 4 bits:
```
Register 0: pins[0-7]   → bits [3:0]=pin0, [7:4]=pin1, ... [31:28]=pin7
Register 1: pins[8-15]
...
Register 19: pins[152-159]
```

Maps to: `io_config` base + 0x000

#### Pulse Control Registers (80 × 32-bit)

Each register holds 2 pins × 16 bits:
```
Register 0: pins[0,1] → bits [15:0]=pin0, [31:16]=pin1
  pin0: [15:8]=pulse_start, [7:0]=pulse_end
  pin1: [31:24]=pulse_start, [23:16]=pulse_end
```

Maps to: `io_config` base + 0x200

### Frequency Selection

| freq_sel | Frequency | MMCM Divider |
|----------|-----------|--------------|
| 0 | 5 MHz | 200 |
| 1 | 10 MHz | 100 |
| 2 | 25 MHz | 40 |
| 3 | 50 MHz | 20 |
| 4 | 100 MHz | 10 |

Maps to: `clk_ctrl` base + 0x00

## Sonoma File Formats Supported

### 1. BIM File (.bim) - XML

Defines device pins, sharing mode, and DUT layout.

```xml
<?xml version="1.0"?>
<BimType type="31" hardware="Sonoma" version="2.0">
    <Layout>
        <Rows>1</Rows>
        <Columns>1</Columns>
    </Layout>
    <Device name="normandy">
        <Pins total="91">
            <Pin share="One DUT" type="Signal">JTAG_TCK</Pin>
            <Pin share="One DUT" type="PmbPS">VDD_CORE</Pin>
            <Pin share="One DUT" type="CorePS">VENG</Pin>
            <Pin share="One DUT" type="ADC">DIODE_TEMP</Pin>
        </Pins>
        <CorePsMonitor>
            <Pin name="VENG" vAdc="VENG_S" iAdc="VENG_I"/>
        </CorePsMonitor>
    </Device>
    <Duts total="1">
        <Dut name="C0">1,2,3,LC1,LC2,CPS1,A0</Dut>
    </Duts>
</BimType>
```

**Pin Types:**
- `Signal` - Digital I/O pin
- `PmbPS` - PMBus power supply
- `CorePS` - VICOR core supply
- `ADC` - Analog measurement

**Share Modes:**
- `One DUT` - Pin connected to single DUT
- `Some DUTs` - Pin shared by subset of DUTs
- `All DUTs` - Pin shared by all DUTs

### 2. MAP File (.map) - Text

Maps tester channels to DUT signals.

```
B13_GPIO0 PWRGD;
B13_GPIO25 JTAG_TCK;
B34_GPIO126 JTAG_TDO;

VOUT1 VOSC;
IOUT1 VOSC_I;
ADC_0 DIODE_TEMP;
XADC_0 VENG_S;
```

**Format:** `<tester_channel> <dut_signal>;`

**Tester Channels:**
- `B13_GPIO0-47` - Bank 13 pins (gpio[0:47])
- `B33_GPIO48-95` - Bank 33 pins (gpio[48:95])
- `B34_GPIO96-127` - Bank 34 pins (gpio[96:127])
- `VOUT1-15` - Voltage outputs
- `IOUT1-15` - Current monitors
- `ADC_0-15` - External ADC channels
- `XADC_0-15` - Internal XADC channels

### 3. TIM File (.tim) - Shell Script

Configures pin types, timing, and clock frequency.

```bash
# Set pattern frequency (5 MHz)
/mnt/bin/linux_xpll_frequency.elf 1 5000000 0 50

# Set pin types
/mnt/bin/linux_pin_type.elf 126 1    # TDO = INPUT (monitor)
/mnt/bin/linux_pin_type.elf 127 1    # HSP_BOOT = INPUT

# Set pulse timing (pin 45 = PULSE with start=50, end=150)
/mnt/bin/linux_Pulse_Delays.elf 45 4 50 150 200
```

**Pin Type Codes:**
| Code | Type | Description |
|------|------|-------------|
| 0 | BIDI | Bidirectional with compare |
| 1 | INPUT | Input only (monitor/compare) |
| 2 | OUTPUT | Output only (drive) |
| 3 | OPEN_C | Open collector |
| 4 | PULSE | Pulse with programmable timing |
| 5 | NPULSE | Inverted pulse |
| 6 | ERR_TRIG | Error trigger output |
| 7 | VEC_CLK | Vector clock output |
| 8 | VEC_CLK_EN | Clock enable output |

## FPGA Register Map

### io_config.v (Pin Configuration)

**Base Address:** 0x4005_0000 (AXI_PIN_CTRL_BASE)

| Offset | Register | Description |
|--------|----------|-------------|
| 0x000-0x04F | pin_type_reg[0-19] | 4 bits/pin × 160 pins |
| 0x200-0x33F | pulse_ctrl_reg[0-79] | 16 bits/pin × 160 pins |

### clk_ctrl.v (Clock Control)

**Base Address:** 0x4008_0000 (AXI_CLK_CTRL_BASE)

| Offset | Register | Description |
|--------|----------|-------------|
| 0x00 | FREQ_SEL | [2:0] frequency selection |
| 0x04 | STATUS | [0] MMCM locked (RO) |
| 0x08 | ENABLE | [0] vec_clk enable |

### axi_fbc_ctrl.v (FBC Control)

**Base Address:** 0x4004_0000 (AXI_FBC_CTRL_BASE)

| Offset | Register | Description |
|--------|----------|-------------|
| 0x00 | CTRL | [0]=enable, [1]=reset, [2]=irq_done, [3]=irq_error |
| 0x04 | STATUS | [0]=running, [1]=done, [2]=error |
| 0x20 | FAST_DOUT | Fast pin drive values |
| 0x24 | FAST_OEN | Fast pin output enables |
| 0x28 | FAST_DIN | Fast pin input states (RO) |

## Firmware Integration

### Loading Configuration

```rust
use std::fs::File;
use std::io::Read;

// In firmware/src/device_config.rs

const IO_CONFIG_BASE: u32 = 0x4005_0000;
const CLK_CTRL_BASE: u32 = 0x4008_0000;

pub fn load_device_config(fbc_data: &[u8]) -> Result<(), Error> {
    // Verify magic
    let magic = u32::from_le_bytes([fbc_data[0], fbc_data[1], fbc_data[2], fbc_data[3]]);
    if magic != 0x00434246 {
        return Err(Error::InvalidMagic);
    }

    // Load timing config
    let freq_sel = fbc_data[8];
    write_reg(CLK_CTRL_BASE + 0x00, freq_sel as u32);

    // Load pin type registers (20 × 32-bit at offset 0x10)
    for i in 0..20 {
        let offset = 0x10 + i * 4;
        let val = u32::from_le_bytes([
            fbc_data[offset], fbc_data[offset+1],
            fbc_data[offset+2], fbc_data[offset+3]
        ]);
        write_reg(IO_CONFIG_BASE + (i * 4) as u32, val);
    }

    // Load pulse control registers (80 × 32-bit at offset 0x60)
    for i in 0..80 {
        let offset = 0x60 + i * 4;
        let val = u32::from_le_bytes([
            fbc_data[offset], fbc_data[offset+1],
            fbc_data[offset+2], fbc_data[offset+3]
        ]);
        write_reg(IO_CONFIG_BASE + 0x200 + (i * 4) as u32, val);
    }

    Ok(())
}
```

### Using Generated C Header

```c
#include "normandy_config.h"

void load_device_config(void) {
    volatile uint32_t *io_config = (uint32_t *)0x40050000;
    volatile uint32_t *clk_ctrl = (uint32_t *)0x40080000;

    // Set frequency
    clk_ctrl[0] = FBC_CONFIG_FREQ_SEL;

    // Load pin types
    for (int i = 0; i < 20; i++) {
        io_config[i] = fbc_pin_type_regs[i];
    }

    // Load pulse control
    for (int i = 0; i < 80; i++) {
        io_config[0x80 + i] = fbc_pulse_ctrl_regs[i];  // 0x200/4 = 0x80
    }
}
```

## GUI Integration

### Reading .fbc Files

The GUI can read `.fbc` files directly using the same format:

```typescript
// In GUI (TypeScript/JavaScript)

interface PinConfig {
  name: string;
  pinType: number;
  pulseStart: number;
  pulseEnd: number;
}

interface DeviceConfig {
  version: number;
  bimType: number;
  pinCount: number;
  freqSel: number;
  pins: PinConfig[];
}

function parseFbcFile(buffer: ArrayBuffer): DeviceConfig {
  const view = new DataView(buffer);

  // Check magic
  const magic = view.getUint32(0, true);
  if (magic !== 0x00434246) {
    throw new Error('Invalid FBC file');
  }

  const config: DeviceConfig = {
    version: view.getUint16(4, true),
    bimType: view.getUint8(6),
    pinCount: view.getUint8(7),
    freqSel: view.getUint8(8),
    pins: []
  };

  // Parse pin types (offset 0x10, 20 registers)
  for (let i = 0; i < 160; i++) {
    const regIdx = Math.floor(i / 8);
    const bitOffset = (i % 8) * 4;
    const reg = view.getUint32(0x10 + regIdx * 4, true);
    const pinType = (reg >> bitOffset) & 0xF;

    // Parse pulse control (offset 0x60, 80 registers)
    const pulseRegIdx = Math.floor(i / 2);
    const pulseReg = view.getUint32(0x60 + pulseRegIdx * 4, true);
    const pulseVal = (i % 2 === 0) ? (pulseReg & 0xFFFF) : (pulseReg >> 16);
    const pulseStart = (pulseVal >> 8) & 0xFF;
    const pulseEnd = pulseVal & 0xFF;

    // Parse name (offset 0x1A0, 16 bytes each)
    const nameOffset = 0x1A0 + i * 16;
    let name = '';
    for (let j = 0; j < 16; j++) {
      const c = view.getUint8(nameOffset + j);
      if (c === 0) break;
      name += String.fromCharCode(c);
    }

    config.pins.push({ name, pinType, pulseStart, pulseEnd });
  }

  return config;
}
```

### Display Helpers

```typescript
function pinTypeName(type: number): string {
  const names = ['BIDI', 'INPUT', 'OUTPUT', 'OPEN_C', 'PULSE',
                 'NPULSE', 'ERR_TRIG', 'VEC_CLK', 'VEC_CLK_EN'];
  return names[type] || 'UNKNOWN';
}

function freqName(sel: number): string {
  const freqs = ['5 MHz', '10 MHz', '25 MHz', '50 MHz', '100 MHz'];
  return freqs[sel] || 'UNKNOWN';
}
```

## Verification Checklist

### 1. Binary Format Verification

- [ ] Magic number correct (0x00434246)
- [ ] Version field matches expected
- [ ] Pin count matches actual configured pins
- [ ] Pin type bits align with FPGA io_config.v expectations
- [ ] Pulse control format matches [15:8]=start, [7:0]=end

### 2. Sonoma Compatibility

- [ ] Parse all .bim files in `Downloads/Bims_extracted/Bims/`
- [ ] Verify pin names extracted correctly
- [ ] Verify BIM type IDs match expected values:
  - normandy = 31
  - iliad = 26
  - syros_v2 = 33
- [ ] Parse .map files with all channel types (B13, B33, B34, VOUT, ADC, XADC)
- [ ] Parse .tim files with all command types

### 3. FPGA Register Verification

- [ ] Pin type register layout matches io_config.v line 276-282
- [ ] Pulse control register layout matches io_config.v line 284-305
- [ ] Frequency selection values match clk_ctrl.v freq_sel encoding
- [ ] Base addresses match fbc_pkg.vh defines

### 4. Firmware Integration

- [ ] Firmware can read .fbc from flash/network
- [ ] Firmware correctly writes to AXI registers
- [ ] Pin configuration takes effect immediately
- [ ] Clock frequency changes work

### 5. GUI Integration

- [ ] GUI can load and parse .fbc files
- [ ] Pin names display correctly
- [ ] Pin types display with human-readable names
- [ ] Frequency displays correctly
- [ ] Can save modified config back to .fbc
